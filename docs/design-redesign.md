## 核心设计决策

### 1. Hero 区域：从"功能堆砌"转向"价值传达"

**改前**：副标题描述技术实现（工作流、PWA），badges 列举功能数量
**改后**：副标题直接说用户能得到什么（30 秒生成方案），badges 强调零门槛（免费、无注册）

**依据**：竞品 Typefully 首屏只传达一个信息——"打开就写"。信息密度降低 = 决策成本降低。

### 2. 新增 "立即开始创作" CTA

**决策**：使用暖色橙金色（`#ffb864`）作为 CTA 色，与品牌冷蓝色 `#59c4ff` 形成互补对比。

**依据**：互补色对比让 CTA 成为视觉焦点。暖色在暗色界面中天然吸引注意力，提升点击欲望。

### 3. 3 步快速引导区

**决策**：在模块目录前插入"填写主题 -> 生成内容 -> 发布到平台"引导

**依据**：解决"15 个模块平铺，用户不知从何开始"的核心问题。新用户看到 3 步就能理解完整流程，降低认知负荷。

### 4. 卡片视觉分层（card-featured）

**决策**：前 3 个模块（创作核心链路）使用加亮边框 + 增强发光效果

**依据**：视觉重量引导用户聚焦核心功能，而非平等对待所有 15 个模块。

### 5. 移动端底部固定导航

**决策**：5 个快速入口（创作/Thread/平台/排版/导航），仅 700px 以下显示

**依据**：移动端用户上下滚动 15 个模块太累，固定底部栏提供随时可达的快速跳转。

### 6. Stagger 动画优化

**决策**：移除 CSS 固定的 nth-child 延迟，改为 JS 动态计算

**依据**：原来只支持 6 个 nth-child，新增 quick-start 后需要灵活的动态延迟。JS 方案更可维护。
