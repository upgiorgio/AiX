## 五、整体建议

### 1. 语气统一
全站整体语气偏「技术文档 + 行业术语」，建议统一为「专业但友好」的风格。具体做法：
- 将"沉淀""飞轮""回流"等互联网黑话替换为日常用语
- 将"主控台""控制台"等开发术语替换为"主页""设置"

### 2. 术语注释策略
建议采用两种方式：
- **高频术语**（Thread、CTA、72h 分发）：在首次出现时用括号加中文解释
- **低频术语**（MVP、JSON、PWA）：使用 `title` 属性做 tooltip

### 3. 按钮文案统一
- "打开 X 发..." 系列统一改为 "去 X 发布..."
- "复制xxx" 系列保持一致
- "导入/导出" 系列保持一致
- 所有主要操作按钮用动词开头

### 4. Card Suite 内部引用清理
card-suite 的 index.html 和 strategy.html 包含大量内部文档引用（"来自 3.1""来自 4.2"），这些对终端用户没有意义，应该移除。

### 5. 中英混杂问题
- "Copy Studio" "Card Designer" "Publish Hub" "Strategy Board" 等英文副标题可以保留但不必要
- 建议统一为中文优先，英文作为品牌辅助
